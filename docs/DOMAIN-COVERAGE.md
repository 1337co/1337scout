# Domain coverage

How 1337scout's discipline maps to the 10 most-used coding domains (by Stack Overflow 2024/2025 usage + GitHub language/tool adoption).

**What "covered" means here ‚Äî read this first.** The kit *disciplines* work; it does **not** add domain capability. A frontier model already writes React, Terraform, and SQL well, and a `python-pro` / `react-specialist` persona is a ceiling-trap (instruction can't lift a model already at its capability ceiling), not a gain ‚Äî so the kit ships none. A domain is "covered" when the kit catches that domain's characteristic *discipline* failure ‚Äî the mistake a capable-but-unsupervised model still makes ‚Äî through a **verifier agent** (which can't edit the work it reviews), a maintained **deterministic scanner** that `/scan-setup` *wires* into the project (you run the tool and act on its findings; the kit sets the gate, it does not execute it), or the mechanical **safety floor**. (`/mcp-setup` likewise *wires* maintained MCP capability servers ‚Äî it points at capability, it does not add it, so it is not itself a catch-mechanism.) That is why the kit stays lean as the base model improves: there is no capability surface to maintain.

**Cross-cutting ‚Äî every domain.** A destructive command or a secret-write is blocked mechanically before it runs (`boundary-guard` ¬∑ `secret-scanner` ‚Äî the `47/47`-caught floor, `0` false positives on 26 benign). Performance regressions go to `@perf-reviewer` (by measurement). A false "done", folding under pushback, and lost context across a long session are held by the always-on discipline kernel (prove-it-done ¬∑ hold-under-pressure) and recorded as machine-readable receipts by the `evidence-ledger` MCP. `/scout` routes any of this by diagnosis, not a hardcoded table.

**The honest limits.** Mobile has no platform-specific element. The mechanical matchers document their own residuals (including hyphenated cloud subverbs, shell quote-obfuscation, variable-indirected targets, some `--no-verify` and sudo / `sh -c`-wrapped forms, and encoded or novel-vendor secrets) ‚Äî the permission deny/ask list is the complementary layer for those, and none of the three layers is a complete sandbox. The kit's bet: disciplining *how* a model works across these domains beats trying to out-capability a model already strong at all of them.
